# Rubyによるデザインパターン
## 基本原則
* 変わるものを変わらないものから分離
* インターフェイスに対してプログラミングし、実装に対して行わない
* 継承より集約
* 委譲、委譲、委譲(delegation)
* YAGNI(You Ain't Gonna Need It) - 必要になるまで作るな
